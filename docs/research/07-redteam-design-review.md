# Red-Team Design/Spec Review (Phase 0, pre-code)

> **Status:** research input, not canon (see [`README.md`](README.md) tier rules).
> **Date:** 2026-06-16. **Method:** adversarial review of the v1 spec docs by an
> AI red-team agent fanning out into four surface-specific passes (crypto/protocol,
> provisioning/recovery, DNS/app-block, lifecycle/transparency) plus direct
> doc reading. No code exists yet — every finding is against the *specification*.
> **Adversary model:** primarily the child trying to escape policy; secondarily a
> malicious parent (stalkerware) and a LAN attacker.
>
> "Covered?" cites a doc file or `GAP` (not addressed, or only the inverse is).
> Severities are relative to the project's fail-closed / no-content /
> anti-stalkerware non-negotiables. Findings are AI-generated and **must be
> human-verified** before driving ADRs or code.

## Top 10 most urgent

1. **R1** — Unauthenticated kid-triggerable 7-day self-wipe (contradiction; three disagreeing specs for an irreversible control). `RECOVERY.md:136`.
2. **C1/C2** — Pledge-breaking content reads: foreground screenshotting + message-text classifier with parent-readable `summary`. Cross the "no content monitoring / no screen recording" non-negotiable. Need an ADR before any code. `LOCAL_AI.md:23,28-31`.
3. **K3** — DNS fails *open* to unfiltered network DNS on crash/cert-fail/OTA — repeatable by a kid, violates fail-closed. `DNS_RESOLVER.md §11`.
4. **K1** — policy_seq rollback not actually prevented (DataStore restore replays old permissive bundle). `CRYPTO.md §8`.
5. **K2** — Event-log tail-truncation: rooted kid drops unsynced incriminating entries; chain can't detect pre-sync deletion. `PROTOCOL.md §4.6`.
6. **B1 + G1 + A1** — App-block escapes: allowlist-vs-blocklist order undefined (clones/Private Space win), `gds.google.com/gmsdrops` hidden browser uncovered in DEFENSES, accessibility-services class not locked down.
7. **N1** — DoH/QUIC/ECH/IP-literal/cronet route around DoT, including inside YouTube — D1 weaker than claimed.
8. **T1/T2** — Time-window holes: cold-boot pre-sync unspecified, 72h-silence disables skew check, device TZ may override signed `tz`.
9. **P1/P2** — Recovery-phrase low-friction wipes: zero-delay emergency decommission + QR that encodes the 24 words.
10. **JC1/SB1/SG1** — Crypto spec defects/contradictions: u64 under 2^53 (update-freeze DoS), authenticated-box vs sealed-box (breaks "child can't read log"), divergent signing-input byte ranges.

## Contradictions / doc fixes needed

- **Decommission trigger** (crit): `RECOVERY.md:136` vs `PARENT_AS_ADVERSARY.md:111,279` + `GRADUATED_PRIVILEGES.md §9`. Resolve via ADR.
- **Content pledge**: `LOCAL_AI.md:23,28-31` vs `KID_TRANSPARENCY.md:108` / `ARCHITECTURE.md:135` / CLAUDE.md non-negotiable.
- **OEM unlock**: `ATTACKS.md:127` vs `ANDROID_COMPAT.md §1/§2`.
- **User-restriction baseline mismatch**: `DEFENSES.md` lists USER_SWITCH/ADD_USER/REMOVE_USER/OEM_UNLOCK as shipped; `ATTACKS.md:98-110` lists the same set under "restrictions to ADD (not in current scaffold)."
- **Event encryption primitive**: anonymous `crypto_box_seal` (`CRYPTO.md §4`/`PROTOCOL.md §6`) vs authenticated `box(...,sender_priv)` + double-ratchet (`STORE_AND_FORWARD.md`).
- **Sealed-box nonce hash**: BLAKE3 (`PROTOCOL.md §6.2`) vs BLAKE2b (`CRYPTO.md §4` + actual libsodium) — literal PROTOCOL impl won't interop; events silently fail to decrypt (fail-open logging).
- **Signature input bytes**: canonical-entry-minus-sig (PROTOCOL/CRYPTO) vs `payload‖prev_hash‖seq` subset (`STORE_AND_FORWARD.md`).
- **Anti-replay counter name/scope**: `policy_seq` (PROTOCOL) vs `cmd_seq` keyed by `(parent_pub, cmd_seq)` (`CRYPTO.md §5`) — keying by parent pubkey means rotation resets the floor (replay-after-rotate).
- **Window clock source**: monotonic (`PROTOCOL.md §2.1` step 6) vs `walletNow()` (`CRYPTO.md §5/§9`) for the same check.
- **Cover-traffic constants**: 2048B/variable (PROTOCOL) vs 4096B/5-min (CRYPTO); stale-mode 60s cadence leaks the device's stale state.
- **05-redteam-deeper proposed DEFENSES #21-28 not in actual `DEFENSES.md`** (accessibility lockdown, Private Space block, gds.google.com filter, Chrome enterprise policy, lock-screen/QS/camera lockdown, PiP teardown).
- **Doc-ref hygiene**: `ATTACKS.md:6-7` points at `C:\src\openwarden-redteam-*.md` *outside* the repo (uncommitted); `FAMILY_MODEL.md:8` + `GRADUATED_PRIVILEGES.md` have broken relative links; `docs/ARCHITECTURE.md` doesn't exist (canonical is repo-root `ARCHITECTURE.md`).

## Adversary-realism caveat

The catalogued threat model (`05-redteam-deeper.md:10`) assumes a **bootloader-locked Pixel 7** baseline. Most high-severity escapes (S1 OEM unlock; root-dependent K1/K2/GF1/TB1) get materially easier on the Tier 2/3 hardware that ADR-001/`ANDROID_COMPAT.md` also target. Docs should state per-tier which guarantees actually hold.
